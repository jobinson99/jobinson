【2012】手机输入法界面布局测评

摘要：

话题： 手机输入法 1   用户交互界面 1  

类别： 贴身移动 创建时间：2012-05-05 13:07:25

说明：

  1. 由于个人时间精力所限，所列并非完整的单款手机输入法的优劣汇总。也就是说，有的问题或者优点是普遍存在的，但我只在某输入法中提到，所以有心的开发者应该逐一对照着每一条，看看是否存在这样的问题或者看看那些可以借鉴
  2. 下文所列输入法完全按测试先后顺序排列
  3. 前面符号所代表含义：
    • ■为个人认为可借鉴的方面
    • □为个人认为是缺陷的方面

讯飞输入法的一些不错功能和不足：

  1. ■手写方面：连写不停笔——超赞！手写和键盘输入不需切换，方便！
  2. □9键布局小键盘布局不好，常用符号不足，这方面其他输入法都做得不错
  3. □英文输入烦琐，切换方式太多
  4. ■符号很全
  5. □编辑部分～复制粘贴等文本操作布局很乱

搜狗输入法

  1. ■界面布局紧凑，当然也就略显界面紧张，但对于触控而言可降低误击率
  2. □选择操作界面竟然没选择起始，删除仅有向左删没向右删
  3. □各个界面部不一致:拼音输入，词语选择，手写，常用标点的位置都不一致
  4. ■列出最近使用的符号

触宝输入法 (本人手机系统自带)

  1. ■复杂操作界面布局合理，复制剪切粘帖放一起，上下页终始选择删除放置有序，没有前删~其他键区有
  2. □选择字才九个，太少，可以十二个
  3. □字母切换上有点复杂，其实一个按钮就够
  4. ■输入时键位变色，同时放大显示输入结果~讯飞只是该键位变色，对于手机用户可能不够直观，因为键区可能被手指挡住了

安卓谷歌拼音输入法

  1. 比较简单，没有九宫格，键位会显得太小，输入的准确度会不足。
  2. 实在太简单基础了，基本满足要求而已，所以就不多说了。

百度手机输入法 5月版

  1. ■常见符号布局合理，都放左侧 符号和表情切换其实可以一个按钮
  2. ■界面初看显得机械，但却是切换最一致的，相对减少界面切换的适应时间和感觉～可能需要个好皮肤～另外，这个键盘键位可以说是可按面积最小的，在拇指运动中，可按面积越大输入可以越准确，综合输入效率可以越高
  3. □输入法顶部列输入法切换，其实那一栏没必要存在，切换可放最右下角，如果真要存在，就放操作提示好了
  4. ■复杂操作布局是所有测试的输入法中最好，但删除方式单一，只能向左删～那个开关灯按钮其实可以放皮肤里
  5. ■界面中文比较完全，适合国人使用

6月版的百度手机输入法，让我有吃苍蝇的感觉了！！！！在百度贴吧上，我还特别对其团队强调不要把事情弄复杂了，结果

  1. ■界面漂亮了点，使用了一些亮黑（虽然是黑色背景，但使用黄色等亮色，使得整个界面看起来亮堂堂的）的透明效果，不错
  2. □复杂化了，添加了很多不实用的功能！！！或者说内嵌了很多不该是输入法的功能，甚至觉得比搜狗还穷凶极恶的赚钱机器一部！
  3. ■把图片、录音、涂鸦、位置等归入输入法，比较有想法
  4. □把破二维码， 语音识别，搜索放入输入法，还当输入法是个专职的快速工具么？当操作系统了吧？虽然可以不启用，但启用的结果是输入法丧失其本身快速反应能力了。

扣扣输入法

  1. ■界面有点智能，但远不够：指在输入时，原来的符号按钮自动变成切分，不够的是其他无法起作用的按钮不同时变，比如键盘切换按钮
  2. □输入法占用界面空间过大，除输入法本身一大块外，输入时还多占拼音一行
